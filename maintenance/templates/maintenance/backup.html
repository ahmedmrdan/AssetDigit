📖 Backup Guide / دليل النسخ الاحتياطي

🇬🇧 English Guide
Which backup to use?
• .db file = Full backup. Use this to completely restore your system.
• JSON = Data export. Good for moving to another system.
• Excel/CSV = For reports and sharing with management.

How often to backup?
• Daily if you have active operations
• Weekly minimum recommended

Where to save?
• USB drive ✅
• Google Drive / OneDrive ✅
• Email it to yourself ✅
• Second PC ✅
🇸🇦 الدليل العربي
أي نسخة احتياطية تستخدم؟
• ملف .db = نسخة احتياطية كاملة. استخدمها لاستعادة النظام بالكامل.
• JSON = تصدير البيانات. جيد للنقل إلى نظام آخر.
• Excel/CSV = للتقارير والمشاركة مع الإدارة.

كم مرة تأخذ نسخة احتياطية؟
• يومياً إذا كانت العمليات نشطة
• أسبوعياً كحد أدنى

أين تحفظها؟
• فلاشة USB ✅
• Google Drive / OneDrive ✅
• أرسلها لنفسك بالبريد الإلكتروني ✅

🔄 How to Restore / كيفية الاستعادة

EN: To restore from a .db backup file:
1. Stop the server (Ctrl+C)
2. Delete the current cmms.db file
3. Copy your backup file and rename it to cmms.db
4. Run python manage.py runserver
5. All your data is back ✅
AR - كيفية الاستعادة من ملف .db:
1. أوقف السيرفر (Ctrl+C)
2. احذف ملف cmms.db الحالي
3. انسخ ملف النسخة الاحتياطية وأعد تسميته إلى cmms.db
4. شغّل python manage.py runserver
5. كل بياناتك عادت ✅
